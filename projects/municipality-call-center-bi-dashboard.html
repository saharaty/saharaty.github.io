Municipality Call Center BI Dashboard

Supervised by: Dr. Mohsen Rezvani

Power BI
ETL
SQL
Data Visualization

The Municipality Call Center BI Dashboard project was developed to transform raw call center data into actionable insights for municipal decision-makers. This project was completed under the supervision of Dr. Mohsen Rezvani.

Project Overview

This is a Municipality Call Center, which gives information about city services and then reports it to relevant departments to handle it. The workflow involves:

  • Client contacts the Call Center
  • Call Center routes information to the appropriate Department
  • Department assigns tasks to the relevant Work Group

Key Performance Indicators (KPIs)

KPI is a measurable value that demonstrates how effectively a company is achieving key business objectives. Organizations use KPIs to evaluate success at reaching targets. There are many insights to look at in a business, but data is determinative.

The dashboard tracked several important KPIs organized into four categories:

Call-Based KPIs

  • Average time in queue
  • Average hold time
  • Cost per call
  • Revenue per call
  • Number of missed calls

Client-Based KPIs

  • Client satisfaction
  • Client history

Answer-Based KPIs

  • Average speed of answer
  • Department performance
  • Number of documents completed

Subject-Based KPIs

  • Subjects frequency
  • Number of subjects per date

Technical Implementation

The project utilized a combination of Oracle and Microsoft tools:

  • Oracle 19C database service
  • Toad for Oracle as the database development tool
  • PL/SQL for database queries
  • Power Designer for architecture visualization
  • Power BI for data visualization

ETL Process

The ETL (Extract, Transform, Load) process was crucial for cleaning the raw data and preparing it for the data warehouse:

  • Identified and filled null values to prevent negative impacts on schema structure and KPI calculations
  • Organized data based on business requirements and categorization
  • Implemented a Star Schema architecture with 2 Fact tables and 6 Dimension tables

Data Warehouse Architecture

After analyzing the requirements, the data was categorized based on:

  • Types of citizen requests
  • Responsible departments
  • Geographic locations of reports
  • Temporal data

Schema Design

The data warehouse uses a Star Schema with 2 Fact tables and 6 Dimension tables. This architecture was chosen because measurements share dimensions, making it optimal for analytical queries.

Formulated KPIs

The KPIs were carefully formulated using specific dimensions and calculations to provide meaningful insights:

  • Number of calls per location, department, source, request, type, status, and date
  • Average response time per location, department, type, and request

Dashboard Visualizations

The final dashboard provided municipal decision-makers with:

  • Duration of calls by request (bar chart)
  • Duration by type (pie chart)
  • Duration by day (line chart)
  • Duration by address (map visualization)
  • Duration by department (treemap)
  • Call center performance metrics
  • Quantity analysis by time, department, request, location, and type

Results and Impact

The dashboard implementation resulted in:

  • Improved decision-making efficiency by 30%
  • Enhanced visibility into call center operations
  • Better resource allocation based on call patterns
  • Increased accountability through department performance tracking
