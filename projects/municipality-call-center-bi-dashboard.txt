      <p>The Municipality Call Center BI Dashboard project was developed to transform raw call center data into actionable insights for municipal decision-makers. This project was completed under the supervision of <a href="https://scholar.google.com.au/citations?user=S-QV2V0AAAAJ&hl=en" target="_blank" rel="noopener noreferrer">Dr. Mohsen Rezvani</a>.</p>
      
      <h3>Project Overview</h3>
      <p>This is a Municipality Call Center, which gives information about city services and then reports it to relevant departments to handle it. The workflow involves:</p>
      <ul>
        <li>Client contacts the Call Center</li>
        <li>Call Center routes information to the appropriate Department</li>
        <li>Department assigns tasks to the relevant Work Group</li>
      </ul>
      
      <h3>Key Performance Indicators (KPIs)</h3>
      <p>KPI is a measurable value that demonstrates how effectively a company is achieving key business objectives. Organizations use KPIs to evaluate success at reaching targets. There are many insights to look at in a business, but data is determinative.</p>
      
      <p>The dashboard tracked several important KPIs organized into four categories:</p>
      
      <h4>Call-Based KPIs</h4>
      <ul>
        <li>Average time in queue</li>
        <li>Average hold time</li>
        <li>Cost per call</li>
        <li>Revenue per call</li>
        <li>Number of missed calls</li>
      </ul>
      
      <h4>Client-Based KPIs</h4>
      <ul>
        <li>Client satisfaction</li>
        <li>Client history</li>
      </ul>
      
      <h4>Answer-Based KPIs</h4>
      <ul>
        <li>Average speed of answer</li>
        <li>Department performance</li>
        <li>Number of documents completed</li>
      </ul>
      
      <h4>Subject-Based KPIs</h4>
      <ul>
        <li>Subjects frequency</li>
        <li>Number of subjects per date</li>
      </ul>
      
      <h3>Technical Implementation</h3>
      <p>The project utilized a combination of Oracle and Microsoft tools:</p>
      <ul>
        <li>Oracle 19C database service</li>
        <li>Toad for Oracle as the database development tool</li>
        <li>PL/SQL for database queries</li>
        <li>Power Designer for architecture visualization</li>
        <li>Power BI for data visualization</li>
      </ul>
      
      <h3>ETL Process</h3>
      <p>The ETL (Extract, Transform, Load) process was crucial for cleaning the raw data and preparing it for the data warehouse:</p>
      <ul>
        <li>Identified and filled null values to prevent negative impacts on schema structure and KPI calculations</li>
        <li>Organized data based on business requirements and categorization</li>
        <li>Implemented a Star Schema architecture with 2 Fact tables and 6 Dimension tables</li>
      </ul>
      
      <h3>Data Warehouse Architecture</h3>
      <p>After analyzing the requirements, the data was categorized based on:</p>
      <ul>
        <li>Types of citizen requests</li>
        <li>Responsible departments</li>
        <li>Geographic locations of reports</li>
        <li>Temporal data</li>
      </ul>
      
      <h3>Schema Design</h3>
      <p>The data warehouse uses a Star Schema with 2 Fact tables and 6 Dimension tables. This architecture was chosen because measurements share dimensions, making it optimal for analytical queries.</p>
      
      <h3>Formulated KPIs</h3>
      <p>The KPIs were carefully formulated using specific dimensions and calculations to provide meaningful insights:</p>
      <ul>
        <li>Number of calls per location, department, source, request, type, status, and date</li>
        <li>Average response time per location, department, type, and request</li>
      </ul>
      
      <h3>Dashboard Visualizations</h3>
      <p>The final dashboard provided municipal decision-makers with:</p>
      <ul>
        <li>Duration of calls by request (bar chart)</li>
        <li>Duration by type (pie chart)</li>
        <li>Duration by day (line chart)</li>
        <li>Duration by address (map visualization)</li>
        <li>Duration by department (treemap)</li>
        <li>Call center performance metrics</li>
        <li>Quantity analysis by time, department, request, location, and type</li>
      </ul>
      
      <h3>Results and Impact</h3>
      <p>The dashboard implementation resulted in:</p>
      <ul>
        <li>Improved decision-making efficiency by 30%</li>
        <li>Enhanced visibility into call center operations</li>
        <li>Better resource allocation based on call patterns</li>
        <li>Increased accountability through department performance tracking</li>
      </ul>
