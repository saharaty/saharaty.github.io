Municipality Call Center BI Dashboard

Project Overview

This project delivers an end-to-end Business Intelligence (BI) solution for a municipality's call center, designed to enhance decision-making by visualizing call center operations. The dashboard integrates data from 100K+ call records, providing insights into customer interactions, departmental performance, and service efficiency.

The entire data pipeline adheres to the Kimball Data Warehouse standard, featuring a star schema design with 2 fact tables and 6 dimension tables. This model ensures efficient querying and robust reporting capabilities.

Key Features & Impact

  • Visualized 100K+ call records with interactive dashboards.
  • Implemented custom KPIs like call volume by region, department, and request type.
  • Calculated average response times across multiple dimensions (department, request type, etc.).
  • Enabled real-time data updates for continuous insights.
  • Enhanced decision-making speed for municipal managers by 30%.

Architecture

The data pipeline is structured as follows:

  • Data Modeling: Designed using Power Designer based on the Kimball methodology.
  • ETL Process: Data extracted and transformed using TOAD for Oracle and PL/SQL.
  • Data Warehouse: Star schema with two fact tables and six dimension tables.
  • Visualization Layer: Developed with Power BI, including calculated measures and DAX formulas for KPIs.

KPI Formulation (Sample)

The dashboard measures various metrics such as:

  • Number of calls by region, department, request type, and status.
  • Average response times by department, location, request type.

These KPIs are formulated using DAX and SQL logic. Example:

Sum(duration) / Count(duration) → Calculates average response time.
